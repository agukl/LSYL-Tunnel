# 目录结构与代码边界

LSYL Tunnel 按客户端、服务端、共享层和部署脚本拆分。当前项目已经从证书身份认证路线简化为账号密码认证路线，因此目录和运维入口也尽量保持短路径。

## 顶层结构

```text
src/
  client/
    cmd/                客户端 CLI / GUI 入口
    cert/               客户端打包用的服务端 TLS 信任文件，只放公开证书
    conf/               客户端默认配置
    gui/                登录界面、托盘值守、窗口控制、本地管理 API
    tunnel/             客户端隧道逻辑、登录探测、转发监听、配置加载
  server/
    assets/             服务端 GUI 和安装器图标资源
    cmd/                服务端 CLI / GUI / Windows 服务入口
    conf/               服务端默认配置
    data/               服务端运行状态，例如未过期封禁 IP
    front/              Web 运维平台页面资源，构建时嵌入服务端 GUI
    gui/                WebView 外壳、本地管理 API、配置读写、Windows 服务控制
    tunnel/             服务端认证、授权、正反向转发、监控、运行事件
  internal/
    protocol/           客户端和服务端共享的握手协议
    transport/          TLS 配置和双向流量复制
    passutil/           密码哈希与校验
    credseal/           客户端本地保存密码的服务端密封凭据
    winservice/         Windows 服务运行封装和按天日志
  cmd/
    lsyl-tunnel-passwd/ 密码哈希工具
    lsyl-tunnel-cert/   服务端 TLS 文件生成工具

deploy/windows/
  build/                构建脚本
  cert/                 服务端 TLS 文件脚本
  run/                  开发运行脚本
  service/              Windows 服务脚本
  app/                  客户端/服务端打包、安装、卸载脚本
  inno/                 Inno Setup 安装器配置和图标输入
  sign/                 代码签名初始化和签名脚本
  test/                 自检脚本

release.cmd             发布主入口：测试、打包、安装器、签名和验证
```

## 客户端边界

客户端负责：

- 读取 `src/client/conf/client.yaml`。
- 校验服务端 TLS 身份。
- 使用用户名密码或短期密封凭据登录服务端。
- 按配置打开本地监听，或激活服务端反向被动端口。
- 在本地连接和服务端隧道之间复制流量。
- GUI 登录成功后进入托盘值守，关闭窗口不等于退出。

客户端不负责：

- 管理用户账号。
- 修改服务端授权。
- 判断最终访问权限。
- 展示复杂 TLS、密钥和服务端诊断细节。

客户端 GUI 的普通用户界面只暴露服务端地址、用户名、密码和连接状态。端口映射、TLS 文件、服务端名称等高级配置由管理员预置。

## 服务端边界

服务端负责：

- 读取 `src/server/conf/server.yaml`。
- 监听隧道入口。
- 校验用户密码或密封凭据。
- 按用户授权决定是否允许访问目标。
- 执行正向目标可达性检查。
- 保存反向代理服务端本地端口配置，客户端激活时尝试监听；端口不可用时记录并由客户端自动重试。
- 维护运行事件、封禁 IP 和监控状态。

服务端不负责：

- 在线注册账号。
- 接收用户上传文件。
- 发放客户端证书。
- 做完整 VPN 路由。
- 主动连接客户端机器。

## 转发方向

`client_to_server`：客户端监听本机入口，服务端连接服务端侧目标。适合用户通过本机端口访问服务端内侧服务。

`server_to_client`：服务端创建被动入口，客户端登录后主动连接服务端并激活该入口。服务端收到访问后通知已连接客户端开数据流，服务端不会主动拨入客户端网络。

反向代理的服务端入口只能是服务端本机回环地址，例如 `127.0.0.1:18080`。同一个反向端口必须全局唯一，并且只能归属一个启用用户。

## GUI 边界

客户端 GUI：

- 内置隧道引擎，不再额外拉起普通客户端进程。
- 默认不注册 Windows 服务。
- 已连接时关闭窗口进入托盘值守。
- 退出客户端才会停止隧道。

服务端 GUI：

- 是本地 Web 运维平台，页面源码在 `src/server/front`。
- 只开放本机 `127.0.0.1` 管理 API。
- 只保留“重启服务”按钮，用于保存配置后重启 `LSYLTunnelServer` Windows 服务。
- 用户认证页只管理用户。
- 转发端口页按已有用户分配端口放通。
- 运行详情页展示封禁 IP、失败诊断、最近事件。

服务端安装器：

- 安装阶段生成自签服务端 TLS 文件，管理员只分发 `server.crt`。
- 默认注册手动启动的 `LSYLTunnelServer` 服务；安装向导确认后可设置为开机自启动。
- 安装、升级和默认卸载保留 `conf`、`certs`、`data`、`logs`。
- Inno 安装器不内置服务端 `.ps1/.cmd` 安装卸载脚本；分发包也不再保留手动安装/卸载入口，只保留安装器生成入口。
- 卸载前停止 GUI/服务进程并删除服务注册，默认只移除程序目录。

## 运行产物

以下目录是构建或运行产物，不是核心源码：

```text
build/bin/client/
build/bin/server/
dist/
tmp/
logs/
certs/
data/
```

根目录 `certs/` 可以由服务端证书脚本重新生成；`src/client/cert/` 是客户端分发包输入目录，会随客户端一起打包和安装。
