# 待做事项

## 已确定的一期优先级

1. 客户端侧：连接失败可理解。
   用户只需要看到直接、可处理的提示，例如服务端不可达、账号密码错误、登录信息过期、本机端口被占用、账号无权访问目标、服务端无法访问目标服务。

2. 服务端侧：授权清晰。
   管理员在“转发端口”页按已有用户分配端口放通关系；“用户认证”页只维护账号本身，不再混入复杂授权字段。

3. 服务端侧：目标可诊断。
   管理员需要能区分失败原因是认证失败、策略拒绝、目标地址错误、目标端口未启动、网络或防火墙不通。

## 当前暂不做

- 客户端端口映射可见性。
- 客户端暴露复杂授权规则。
- 客户端展示过多 TLS、密钥、目标诊断细节。
