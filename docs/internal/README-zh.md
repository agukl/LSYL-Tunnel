## 这一层文档的定位

如果你现在要做的是下面这些事情，再进入 `docs/internal/`：

- 调整源码结构
- 理解 client/server/internal 的边界
- 研究项目在网络路径中的位置
- 做正式发布或签名验收
- 做长期归档或版本交接

如果你现在只是想：

- 部署服务端
- 给用户发客户端
- 看懂整体流程
- 改配置和排障

优先回到 [上一级文档入口](../README-zh.md)。
